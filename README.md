Inventory Management System (Blazor)

Inventory — приложение Blazor, для автоматизации учета, контроля и управления оборудованием, выданным сотрудникам или подразделениям.
🚀 Основная идея

Система предоставляя удобный интерфейс для отслеживания жизненного цикла каждой единицы техники: от закупки до списания или выдачи конкретному лицу.
✨ Основные возможности

    Учет оборудования: Регистрация ТМЦ с указанием серийных номеров, характеристик и состояния.

    Управление выдачей: Фиксация факта передачи оборудования сотруднику (кто, когда и что получил).

    История перемещений: Полный лог того, где находилось устройство ранее.

    Статусы активов: Быстрый фильтр по состояниям (в наличии, выдано, в ремонте, списано).

    Поиск и фильтрация: Мгновенный поиск по серийному номеру, модели или пользователю.

    Адаптивный интерфейс: Удобная работа как с десктопа, так и с мобильных устройств (благодаря Blazor + Bootstrap/MudBlazor).

🛠 Технологический стек

    Frontend: Blazor (Server или WebAssembly)

    Backend: .NET / ASP.NET Core

    Database: Entity Framework Core (SQL Server / PostgreSQL / SQLite)

    UI Library: (Например, MudBlazor или Ant Design — уточните, если используете)

📦 Структура проекта

    Inventory.Shared — общие модели данных и DTO.

    Inventory.Client — клиентская логика и UI компоненты.

    Inventory.Server — API контроллеры и работа с базой данных.

⚙️ Установка и запуск

    Клонируйте репозиторий:
    Bash

    git clone https://github.com/igmo71/Inventory.git
    cd Inventory

    Настройте строку подключения:
    Отредактируйте файл appsettings.json в серверном проекте, указав путь к вашей базе данных.

    Обновите базу данных:
    Bash

    dotnet ef database update

    Запустите проект:
    Bash

    dotnet run --project Inventory/Server
